---
title: Quickstart
description: Install apidiom and generate agent tools from any OpenAPI spec.
---

## Install

Requires Node.js 18+.

```bash
npm install -g apidiom
```

Or use without installing:

```bash
npx apidiom generate mcp stripe --output stripe-mcp.js
```

## List built-in services

```bash
apidiom generate mcp --list
```

You can also pass a local OpenAPI JSON/YAML file or a URL.

## Generate an MCP server

```bash
apidiom generate mcp ./openapi.yaml --list
apidiom generate mcp ./openapi.yaml --include listPets --output pets-mcp.js
node pets-mcp.js
```

Use `--include operationId` or `--tag name` for large specs.

The generated file is standalone JavaScript. It speaks MCP over stdio and uses
Node's built-in `http` and `https` modules for API calls.

## Generate JSON tool schemas

```bash
apidiom generate schema ./openapi.yaml --format openai --output tools.json
apidiom generate schema ./openapi.yaml --format anthropic
```

Schema export writes only tool names, descriptions, and input schemas. It does
not encode auth. Keep auth, retries, and execution behavior in your own runtime
code when using raw SDK schemas.

Real public spec smoke checks used during development:

```bash
apidiom generate mcp \
  https://raw.githubusercontent.com/OAI/OpenAPI-Specification/main/examples/v3.0/petstore.yaml \
  --include listPets \
  --output petstore-mcp.js

apidiom generate schema \
  https://raw.githubusercontent.com/OAI/OpenAPI-Specification/main/examples/v3.0/petstore.yaml \
  --format openai \
  --include listPets \
  --output petstore-tools.json
```

Expected generated tool name from that spec: `list_pets`.

## Use a built-in service

```bash
apidiom generate schema stripe --format anthropic --tag PaymentIntents --output stripe-tools.json
apidiom generate mcp github --tag repos --output github-mcp.js
```

Some APIs require auth for runtime calls. Generated MCP servers read supported
auth values from environment variables and print a clear error when required
values are missing.

## Filter generated tools

```bash
apidiom generate mcp ./openapi.yaml --tag pets --output pets-mcp.js
apidiom generate schema ./openapi.yaml --format openai --include listPets
```

Use tags to keep whole resource groups. Use includes when you already know the
operation IDs you want.
