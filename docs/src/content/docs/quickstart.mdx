## Install

On Windows:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -e ".[dev]"
```

On macOS/Linux:

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install -e ".[dev]"
```

For the primary codegen path, install Java and `openapi-generator-cli`. If Java
is not available, `apidiom` can use its no-Java fallback:
`datamodel-code-generator` for pydantic models plus a minimal `httpx` wrapper
template.

## Generate from a clean OpenAPI spec

```bash
apidiom generate ./openapi.yaml --output client.py --codegen builtin
```

The default input kind is `auto`. A clean JSON or YAML OpenAPI spec is detected
as `openapi`.

## Generate an MCP server

```bash
apidiom generate mcp ./openapi.yaml --list
apidiom generate mcp ./openapi.yaml --include GET:/pets --check --output pets_mcp.py
python pets_mcp.py
```

Use `--include METHOD:/path/{param}` or `--tag name` for large specs.
`--check` compiles the generated server and prints the exported tool count
without calling the API.

## Generate other agent tool formats

```bash
apidiom generate langchain ./openapi.yaml --tag pets --output pet_tools.py
apidiom generate schema ./openapi.yaml --format openai --output tools.json
apidiom generate schema ./openapi.yaml --format anthropic
```

Use multiple specs when one agent needs tools from more than one API:

```bash
apidiom generate mcp ./stripe.yaml ./github.yaml --output unified_mcp
```

LangGraph uses LangChain tools directly, so the same output works for both:

```python
from pet_tools import list_pets

tools = [list_pets]
```

Real public spec smoke checks used during development:

```bash
apidiom generate langchain \
  https://raw.githubusercontent.com/github/rest-api-description/main/descriptions/api.github.com/api.github.com.json \
  --include GET:/repos/{owner}/{repo} \
  --output github_tools.py

apidiom generate schema \
  https://raw.githubusercontent.com/stripe/openapi/master/openapi/spec3.json \
  --format anthropic \
  --include POST:/v1/payment_intents \
  --output stripe_tools.json
```

Expected generated tool names from those specs: `repos_get` and
`post_payment_intents`.

If you only know an API base URL, apidiom tries common OpenAPI locations:

```bash
apidiom generate mcp https://api.example.com --output api_mcp.py
```

For sparse specs, enrich generated tool descriptions with an LLM provider:

```bash
apidiom --provider ollama generate mcp ./openapi.yaml --enrich-docs --output api_mcp.py
```

## Generate from messy docs

```bash
apidiom generate https://docs.example.com/api \
  --provider ollama \
  --output client.py \
  --codegen builtin
```

Messy docs are detected as `unstructured`. Use `--input-kind openapi` or
`--input-kind unstructured` only when you need to override detection.

Use `ollama` for private docs. Use `gemini` only for public docs where the
privacy tradeoff is acceptable.

## Check a provider

```bash
apidiom check --provider ollama
apidiom check --provider gemini
```
