# OpenAPI → MCP Generator

## Pipeline

```
Source (service name | file path | URL)
  ↓ resolveSource()          ts/registry.ts
  ↓ fetchSpec()              ts/ingest/fetch.ts  — HTTP/local + resolveRefs
  ↓ parseOpenAPI()           ts/ingest/parse.ts  — doc → APIModel
  ↓ extractAuth()            ts/auth.ts          — authSchemes → AuthConfig[]
  ↓ buildToolMetadata()      ts/generate/tools.ts
  ↓ generateMCPServer()      ts/generate/mcp.ts  → CJS JS string
```

## Output: standalone CJS JS file

The generated file has **no npm dependencies** and runs directly with `node server.js`.

Structure of the generated output:
```
"use strict";
// Generated by apidiom …

const https = require("https");
const http  = require("http");

// --- Auth ---                 (omitted when no auth schemes)
const STRIPE_BEARER_TOKEN = process.env.STRIPE_BEARER_TOKEN;
if (!STRIPE_BEARER_TOKEN) { … process.exit(1); }

// --- HTTP helper ---
function _request(method, url, headers, body) { … }   // rejects on non-2xx

// --- Tools ---
var _TOOLS = [ { name, description, inputSchema, _tags, call } … ];

// --- MCP Protocol ---         (flat mode: exposes _TOOLS directly)
// OR
// --- Search scorer ---        (search mode: exposes search_tools + call_tool)
// --- MCP Protocol (search mode) ---
```

## Modes

| Mode | tools/list returns | When chosen |
|------|--------------------|-------------|
| flat | all tools from _TOOLS | toolCount ≤ maxTools (default 40) |
| search | search_tools + call_tool meta-tools | toolCount > maxTools (auto) or --mode search |
| auto | picks flat or search based on toolCount | default |

## Auth wiring

| Scheme type | Generated env var | Header |
|-------------|-------------------|--------|
| apiKey in header | {SERVICE}_{KEY_NAME} | key name from spec |
| apiKey in query | {SERVICE}_{KEY_NAME} | appended as query param |
| http bearer | {SERVICE}_BEARER_TOKEN | Authorization: Bearer {value} |
| http basic | {SERVICE}_BASIC_TOKEN | Authorization: Basic {value} |
| oauth2, openIdConnect | skipped (warning emitted) | — |

## Tool naming

normalizeToolName(operationId):
1. Strip version segments: GetV1ChargesId → GetChargesId
2. camelCase/PascalCase → snake_case: GetChargesId → get_charges_id
3. Deduplicate collisions with _2, _3 suffix

--group-by-tag prefixes with tag: pets__list_pets

## $ref resolution

resolveRefs (called by fetchSpec) handles:
- $ref on parameter objects → resolved inline
- $ref on requestBody objects → resolved inline
- Does NOT recurse into parameter schemas (deliberate: avoids OOM on Stripe-scale specs)
- Only local (#/) refs are resolved; external refs pass through

## MCP protocol implemented

- initialize → responds with protocolVersion: "2024-11-05"
- notifications/initialized → no-op
- tools/list → flat: all tools; search: [search_tools, call_tool]
- tools/call → calls tool, returns content: [{type:"text", text: JSON.stringify(result)}]; non-2xx HTTP responses emit isError: true

## Search mode scorer

The TypeScript scoreTools in ts/generate/search.ts and the inlined _scoreTools JS in the generated output implement the same algorithm. Keep them in sync manually when changing scoring weights.

Scoring per query token:
- Name exact token match: +10
- Name substring match: +5
- Description exact token: +2
- Description substring: +1
- Tag exact token: +3

Final score multiplied by (0.5 + 0.5 × termsCovered/totalTerms) for coverage ratio.

## Extending the generator

- Add a new output format → new file under ts/generate/, new generate <format> command in ts/cli.ts, new test file under ts-tests/generate/.
- Never add Python code or Python dependencies. The Python archive lives in archive/python/ and is not active.
